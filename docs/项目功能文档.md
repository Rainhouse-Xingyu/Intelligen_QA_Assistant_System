# Coze智能助手系统 - 详细项目功能文档

本项目是一个结合了前端UI展示、后端业务逻辑、数据安全控制以及大模型（LLM）编排（通过Coze）的智能问答与学业帮扶系统。本文档详述了系统六大核心模块的功能细化、技术落地方案及数据库表结构映射。

---

## 1. 基础运行模块
该模块是系统的底座，负责权限管控、路由分配及多端（Web、公众号）界面的响应。

### 1.1 系统底层服务与认证
- **功能细节**：
  - 支持多角色登录：学生、教师、系统管理员。
  - 用户鉴权与角色路由控制：基于 JWT（JSON Web Token）实现无状态鉴权，系统提供统一的登录和退出接口。
- **技术实现**：利用 Java SpringBoot 结合拦截器（Interceptor）实现 RBAC 权限校验。
- **关联数据表**：`sys_user`（系统用户表，存储角色、学号/工号、密码哈希等信息）。
- **前端工作**：基于 Vue3 + Tailwind CSS / Element Plus 编写登录页及师生端、管理端的响应式框架，同时融入定制化元素确保三大原则（层次感、和谐感、一致性）。

### 1.2 大连东软信息学院一网通 (SSO) 集成
- **功能细节**：
  - 作为校内应用的入口，预留并接入校园“一网通”单点登录（SSO）能力。
  - 学生或教职工在一网通授权后，直接免密进入本系统，无需二次注册或输入账密。
- **技术实现**：
  - 后端对接校园 SSO 认证服务器（通常为 CAS 或 OAuth2 协议）。
  - 根据回调的 Token 换取校园统一身份标识（并在数据库绑定映射）。
- **关联数据表**：`sys_user` 表中预留 `campus_sso_id` 字段并建立唯一索引，绑定校园身份。

### 1.3 微信公众号平台集成
- **功能细节**：
  - 免密登录：通过 OAuth2 获取微信用户 OpenID 并与系统 `sys_user` 绑定。
  - 消息即时交互：公众号后台接收学生提问，路由给系统后端进行 AI 问答后的回复下发。
- **技术实现**：对接微信公众平台开发 API，后端处理 XML/JSON 消息解析与事件路由。
- **前端工作**：在 H5 端（微信内环境）调试页面兼容性，可调用微信原生功能（如果适用）。

## 2. 智能问答模块
系统最为高频使用的交互部分，具备多模态、可转人工、懂业务分发的特性。

### 2.1 Coze API 对接与意图网关
- **功能细节**：前端发起提问后，后端请求封装、透传给 Coze 平台的大模型进行推理，再将回复响应给前端（支持流式/SSE返回以提升体验）。
- **技术实现**：后端通过 RestTemplate/WebClient 封装 Coze OpenAPI 接口交互。
- **关联数据表**：`chat_session` (会话记录)、`chat_message` (具体问答文本/意图标记)。

### 2.2 语音与文字双模态交互
- **功能细节**：除打字外，用户可直接按住说话，也可要求系统将文字答案转为语音播报。
- **技术实现**：前端将音频数据流（Blob/File）上传，后端统一音频编码后请求外部/Coze语音插件。
- **前端工作**：编写录音输入（Recorder.js 等）与原生 HTML5 Audio/AVKit（若适配 Apple 系原生体验）播报组件。

### 2.3 人工介入与闭环处理
- **功能细节**：
  - 用户触发“转人工”后，系统通知空闲的管理端客服。
  - 客服接入后，通过平台直接下发回复，结束后退回给 AI 托管状态。
- **技术实现**：后端基于 WebSocket 建立双向通信的实时聊天室及基础排队派单逻辑。
- **前端工作**：学生端开发工单状态提示；管理端开发“客服工作台”面板接收即时 WebSocket 消息。

### 2.4 业务联动自动推送 API
- **功能细节**：当用户提问包含选课、重修、退学等教务关键字时，系统在给出解答的同时，底部自动附加教务处对应负责老师的座机号码。
- **技术实现**：通过 AI 返回的业务意图或关键字匹配，后端从数据库调出教师联系方式一并随消息下发。
- **关联数据表**：`biz_contact` (业务线联系方式表)。

## 3. 知识库管理模块
供管理人员上传本校考务政策与办事指南，以便 AI 从中检索生成精准回答。

### 3.1 常见文档上传与内容解析（核心）
- **功能细节**：
  - 管理员在后台上传 Word (.docx) 和 Excel (.xlsx) 包含历史的 Q&A 数据。
  - 系统异步解析文本并拆分为有效词条入库。
- **技术实现**：基于 Apache POI 库读取 Word 段落与 Excel 行列，清洗后映射为结构化 QA 数据。
- **关联数据表**：`kb_document` (原始文件表)。

### 3.2 问答词条在线编辑
- **功能细节**：支持后台列表进行检索，管理人员可对已解析或新增的问答进行修改、禁用操作。
- **技术实现**：SpringBoot 开发标准的 RESTful CRUD 接口。
- **关联数据表**：`kb_qa_entry` (问答词条内容表)。
- **前端工作**：基于 Vue3 的表格页（Table）与表单（Form）。

## 4. 数据统计与推送模块
沉淀数字化资产，反馈给管理者甚至反向帮助学生。

### 4.1 常见咨询分析与拦截
- **功能细节**：统计一周、一月内问得最多的词汇和话题。
- **技术实现**：系统在用户发起提问时运用日志拦截器进行埋点计数。
- **关联数据表**：`stat_hot_question` (提问频次统计表)。
- **前端工作**：结合 ECharts 展示“热词云”、“趋势折线图”的大屏界面。

### 4.2 常见问题主动推送服务
- **功能细节**：后台服务按一定频率抽取高频问题，当用户刚登录时，以“跑马灯/猜你想问”抛出预判。
- **技术实现**：Spring Boot 定时任务 (Cron Job) 刷新常见问题缓存，通过 WebSocket 下推。

### 4.3 未识别问题汇总反馈
- **功能细节**：对于问答中 AI 识别度低导致回答“抱歉，我不清楚”的问题自动留存。
- **技术实现**：提问识别度检测兜底入库。
- **关联数据表**：`unrecognized_query` (未识别问题库)。

## 5. 学业帮扶模块
本系统最具人文关怀的模块，深度介入学生的学期成长周期。

### 5.1 学业指标存储与预警接口
- **功能细节**：记录学生的成绩表现（GPA、挂科）、性格评测等多维数据，设定阈值作为风险识别依据。一旦进入学业预警范围，系统根据现状生成对应方案。
- **关联数据表**：`student_profile` (学生画像基础表)。

### 5.2 学生画像标签系统与AI共情
- **功能细节**：为不同学生打上诸如“考研压力大”、“偏内向”、“时间管理弱”的脱敏标签，在请求 Coze Agent 心理倾诉时作为上下文 Prompt，增强机器的“共情”质量。
- **技术实现**：结合本地画像库构造定制化的系统级 Prompt。

### 5.3 帮扶成效报告自动化生成
- **功能细节**：学期末，根据数据变动或AI的长期交流日志总结出 PDF 格式的“学生成长及成效报告”。
- **技术实现**：Java 开源库 (如 iText 或 PDFBox) 根据预设的业务数据模板结构自动化填充内容生成文件。
- **关联数据表**：`academic_warning_record` (学业预警与相关帮扶存档)。

## 6. 安全合规模块 (重点底线)
高校数据不能直接裸奔于云端模型，必须强脱敏。

### 6.1 数据脱敏拦截器（重难点核心）
- **功能细节**：系统不能外传学号或姓名。与大模型交互必须使用类似 UUID 的 `masking_id` 替代身份。任何出现在文本里的邮箱、手机号等也需正则脱敏。
- **技术实现**：
  - 通过 SpringBoot AOP 或者外部请求的全局 Interceptor 机制。
  - 将出栈到 Coze 前的字符串作正则脱敏，处理进栈的请求将 `masking_id` 反向映射回内部日志，对大模型做到 100% “零隐私知晓”。
- **前端工作**：界面同样需遵守权限控制规则，列表展示涉及学生信息须加星号 (`***`)。

# 智能问答辅助系统 — 前端功能文档 v0.1

> 本文档基于 `backend/API接口文档.md` 分析整理，梳理前端各角色功能模块与页面应包含的内容。

---

## 1. 角色定义

| 角色 | 标识 | 说明 |
|------|------|------|
| 学生 | `student` | 系统的核心服务对象，通过 AI 对话获取学业、心理、校园生活等方面的智能问答服务 |
| 客服 | `admin` / `staff` | 人工客服坐席，接管 AI 转人工的会话，与用户实时文字沟通 |
| 教务老师/辅导员 | `admin` / `staff` | 管理知识库、查看热点数据、处理未识别问题、生成学业帮扶策略与报告 |
| 系统管理员 | `admin` | 最高权限，涵盖客服 + 教务老师全部功能，并可管理未识别问题状态 |

> **说明**: 后端 `getUserInfo` 接口返回的 `role` 字段目前已知值包括 `student`。管理员类角色的具体 role 值需与后端确认。以下按"学生端"和"管理端"两大视角划分功能。

## 3. 通用功能（所有角色）

### 3.1 登录认证

| 功能点 | 对应 API | 说明 |
|--------|----------|------|
| 账号密码登录 | `POST /api/auth/login` | 输入用户名+密码，获取 JWT Token |
| 一网通 SSO 登录 | `POST /api/auth/sso/callback` | 携带 SSO 凭证换取系统 Token |
| 微信 OAuth 登录 | `GET /api/wechat/login` | 微信授权回调，传入 code 换取系统 Token |
| 获取用户信息 | `GET /api/auth/info` | 登录后获取当前用户角色、用户名等信息 |
| 登出 | `POST /api/auth/logout` | 退出登录，注销 Token |

**页面内容**：
- 登录表单 (用户名 + 密码)
- "一网通登录" 按钮 → 跳转学校统一认证页面
- "微信登录" 入口 → 展示公众号二维码
- 登录失败提示、Token 过期自动跳转登录页
- 根据返回的 `role` 字段自动路由到学生端或管理端

## 4. 学生端功能

### 4.1 AI 智能问答对话

**对应 API**：
| 接口 | 路径 |
|------|------|
| 发送文本消息 | `POST /api/chat/text` |
| 发送语音消息 | `POST /api/chat/voice` |
| 获取推荐问题 | `GET /api/chat/suggested-questions` |
| 申请转人工 | `POST /api/chat/transfer-to-human` |
| 上报未识别回答 | `POST /api/chat/report-unrecognized` |

**页面内容**：
- **对话区域**: 聊天气泡形式展示用户提问与 AI 回答，支持滚动加载历史记录
- **输入区**: 底部固定文本输入框 + 发送按钮
- **语音输入**: 麦克风按钮 → 录音 → 调用语音接口发送
- **推荐问题气泡**: 初次进入 / 无对话时展示热门的 Suggestion 问题（来自 2.7 接口），点击即可快速提问
- **点踩/反馈按钮**: 当 AI 返回兜底话术（"我不知道"类）时，显示"点踩反馈"按钮，点击后调 2.8 上报
- **转人工按钮**: 始终可见的"转人工客服"按钮，点击后调 2.4，后台排队等待人工接入

### 4.2 心理咨询

**对应 API**：
| 接口 | 路径 |
|------|------|
| 心理咨询工作流 | `POST /api/chat/psychological` |

**页面内容**：
- 与对话页风格类似的聊天界面（可复用对话组件）
- 输入学生反馈信息（情绪描述、困扰等），调用心理安抚工作流
- AI 返回安抚指导 / 话术回复
- 可考虑增加情绪状态标记（如压力等级展示，若后端返回）

### 4.3 个人中心

**页面内容**：
- 显示当前用户基本信息（用户名、角色等，来自 `GET /api/auth/info`）
- 操作入口：退出登录

## 5. 管理端功能

### 5.1 数据看板 / 工作台

**对应 API**：
| 接口 | 路径 |
|------|------|
| 热点提问分析 | `GET /api/stat/hot-questions` |

**页面内容**：
- **热点问题词云**: 使用 ECharts 词云图展示近 N 天的热点提问关键词及其频率
- **筛选条件**: 时间范围选择（最近 7 天 / 14 天 / 30 天），展示数量调整（Top 10 / 20 / 50）
- **数据卡片**: 总提问数、活跃用户数、未处理问题数等统计摘要（如后端支持）
- 作为管理端登录后的默认首页

### 5.2 知识库管理

**对应 API**：
| 接口 | 路径 |
|------|------|
| 上传并解析文件 | `POST /api/kb/upload` |
| 查询问答词条列表 | `GET /api/kb/entries` |
| 手动新增问答 | `POST /api/kb/entries` |

**页面内容**：

**5.2.1 知识库词条列表页**
- 表格展示所有已入库的 Q&A 词条（Question + Answer）
- 支持搜索/筛选（前端本地过滤或后续扩展后端分页查询）
- 每条词条显示：问题、答案摘要、创建时间

**5.2.2 新增词条**
- 表单：问题输入框 + 答案输入框（文本域）
- 提交按钮 → 调用 `POST /api/kb/entries`

**5.2.3 批量导入**
- 上传 .xlsx 文件（拖拽上传区域 or 文件选择器）
- 上传进度提示
- 上传成功后刷新列表

### 5.3 人工客服台

**对应 API**：
| 接口 | 路径 |
|------|------|
| 客服回复用户 | `POST /api/chat/admin/reply` |
| 结束会话交还 AI | `POST /api/chat/admin/finish` |

**页面内容**：
- **会话列表**（左侧栏）: 展示当前排队 / 进行中的人工会话，每项显示学生信息 + 等待时长
- **对话区**（右侧主区域）: 选中某个会话后，展示该会话的历史对话记录
- **输入区**: 底部文本输入框 + 发送按钮（调用 2.5 回复）
- **结束会话按钮**: 点击后调 2.6，将用户交还给 AI，该会话从待处理列表中移除
- **状态标识**: 等待中 / 接待中 / 已结束

### 5.4 未识别问题管理

**对应 API**：
| 接口 | 路径 |
|------|------|
| 未识别问题列表 | `GET /api/admin/unrecognized/list` |
| 更新问题状态 | `POST /api/admin/unrecognized/update-status` |

**页面内容**：
- **表格列表**: 分页展示未识别问题，包含列：问题内容、提问时间、当前状态（未处理 / 已处理 / 已补充到知识库等）
- **筛选**: 按处理状态过滤（全部 / 未处理 / 已处理）
- **操作**: 
  - 查看问题详情
  - "标记为已处理" 按钮 → 调用 4.3 更新状态
  - 可快速跳转到知识库新增词条页面（预填该问题），将未识别问题补充为正式 Q&A
- **分页器**: 支持切换页码和每页条数

### 5.5 学业帮扶

**对应 API**：
| 接口 | 路径 |
|------|------|
| 生成帮扶策略 | `POST /api/academic/generate-warning` |
| 更新学生画像 | `POST /api/academic/profile/update` |
| 生成帮扶 PDF 报告 | `POST /api/academic/generate-pdf-report` |

**页面内容**：

**5.5.1 生成帮扶策略**
- 表单：选择/输入学生 ID + 选择学期范围
- 提交后展示 AI 生成的学业预警干预记录与帮扶策略详情
- 策略结果展示区域（成绩分析、挂科风险、建议措施等结构化信息）

**5.5.2 学生画像数据管理**
- 表单：录入或导入学生画像和成绩数据（挂科、GPA 等字段）
- 提交后调用 5.2 接口更新

**5.5.3 帮扶报告管理**
- 帮扶记录列表：展示历史帮扶记录（可与 5.5.1 生成的记录关联）
- 每条记录提供"生成 PDF 报告"按钮 → 调用 5.3 → 返回可下载 URL
- 点击下载链接在新窗口打开 PDF 或触发浏览器下载

### 5.6 个人中心

**页面内容**：
- 显示管理员信息（同学生端 4.3）
- 退出登录

## 6. 路由与权限控制

| 路由路径 | 页面 | 所需角色 |
|----------|------|----------|
| `/login` | 登录页 | 未登录用户 |
| `/student/chat` | AI 问答对话 | `student` |
| `/student/psychological` | 心理咨询 | `student` |
| `/student/profile` | 学生个人中心 | `student` |
| `/admin/dashboard` | 数据看板 | `admin` |
| `/admin/kb/list` | 知识库列表 | `admin` |
| `/admin/kb/add` | 新增知识库词条 | `admin` |
| `/admin/cs` | 人工客服台 | `admin` |
| `/admin/unrecognized` | 未识别问题管理 | `admin` |
| `/admin/academic/warning` | 生成帮扶策略 | `admin` |
| `/admin/academic/profile` | 学生画像管理 | `admin` |
| `/admin/academic/reports` | 帮扶报告管理 | `admin` |
| `/admin/profile` | 管理端个人中心 | `admin` |

**鉴权策略**：
- 前端路由守卫：未登录 → 跳转 `/login`
- 角色路由守卫：`student` 无法访问 `/admin/*`，`admin` 可访问自身及学生端页面（便于管理员测试对话功能）
- Token 存储于 localStorage，每次请求通过 `Authorization` Header 携带

## 9. 待确认事项

1. **管理员角色细分**: 后端 `role` 字段是否区分 `admin`、`counselor`（辅导员）、`teacher`（教务老师）等子角色？如是，某些管理功能可能需要做权限细分。

5. **微信公众号登录**: 前端是否需要内嵌公众号网页（JS-SDK），还是仅作为 Web 端登录的一个通道？

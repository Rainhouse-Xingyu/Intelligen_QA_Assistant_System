# 智能问答助手系统内嵌 AI 模型方案

## 第一阶段：知识库导入

**管理员上传文件格式：**
- Word
- Excel
- PDF

**技术栈与依赖配置：**

- **Word / Excel 解析：** 使用 `Apache POI`
  ```xml
  <dependency>
      <groupId>org.apache.poi</groupId>
      <artifactId>poi-ooxml</artifactId>
  </dependency>
  ```
- **PDF 解析：** 使用 `PDFBox`
  ```xml
  <dependency>
      <groupId>org.apache.pdfbox</groupId>
      <artifactId>pdfbox</artifactId>
  </dependency>
  ```

## 第二阶段：数据清洗

- **使用模型：** Qwen3-0.6B
- **核心作用：** 自动提取非结构化文档内容并转为主流知识库格式。

**提示词（Prompt）：**
> 请将以下通知转换为 FAQ 格式：  
> 问题：  
> 答案：

**生成示例：**
> **Q:** 补考什么时候开始？  
> **A:** 2026年6月15日开始。

## 第三阶段：知识切片

**切片原则：**
- ❌ **避免使用：** 机械式的字数截断（如：直接按 500 字切一块）。
- ✅ **推荐使用：** 按语义切块（如：一个问题对应一个 Chunk）。

**包含内容的 Chunk 示例：**
- 补考时间
- 缓考申请
- 重修流程

**存储映射：** 存入 `kb_qa_entry` 库表中。

## 第四阶段：向量化

- **使用模型：** BGE-small-zh-v1.5
- **核心作用：** 问题 ——> 向量转换特征。

**执行示例：**
输入一段文本文本：`“挂科怎么办”` 
转换得出：`[0.12, 0.35, ...]`（768维向量）。

## 第五阶段：向量数据库

- **使用组件：** Milvus
- **数据存储基本结构模型：**
  - 向量信息 (Vector)
  - 问题内容 (Question)
  - 答案内容 (Answer)
  - 分类标签 (Category)

**存储记录示例：**
```yaml
ID: 1001
Question: 补考时间
Vector: [0.12, 0.35, ...]
Answer: 2026年6月15日
```

## 第六阶段：学生提问

**业务流程：**
学生输入口语化问题（例如：`“挂了咋办”`），由系统接收。问题数据直接进入并记录在业务表 `question_raw`。

## 第七阶段：问题改写

学生提问往往缺乏规范性或比较口语化，需要通过模型进行意图重写。

- **假设情景：** 学生输入 `“绩点炸了”`
- **改写目标：** 标准教务提问词汇 `“绩点过低会有什么影响”`

- **使用模型：** Qwen3-0.6B
- **提示词（Prompt）：**
> 请将学生问题改写成标准教务问题。

## 第八阶段：模块分类

通常学生入口已默认一级分类（必选操作，如：考务通知、教学运行、学业帮扶等），改写后将进一步通过模型自动化标记归属**二级分类**，例如：`补考`，`重修`，`毕业`，`选课` 等。

- **使用模型：** MacBERT
- **核心作用：** 精准过滤库区范围，从而**缩小检索范围**。

## 第九阶段：向量检索

**检索核心链路：**
1. 获取意图对应的问题：`“挂科怎么办”`。
2. 调用向量化模型：`BGE`。
3. 连接进入：`Milvus` 数据库发起相似度对比检测。
4. 返回命中集合：召回粗排 `Top 20`。
   - *（返回示例：匹配到 `“补考管理办法”`，相似度得分为 `0.91`）*。

## 第十阶段：重排序 (Reranker)

- **使用模型：** bge-reranker-large
- **核心运作流：** 
  提取粗排结果池 `Top 20` ➡️ 通过注意力机制精准计算重新排序 ➡️ 输出得分最高的 `Top 3` 提供给大模型作为上下文。

## 第十一阶段：命中判断

结合分数建立置信度阈值阻断机制，直接采用代码规定执行方案：

```java
if(score >= 0.85){
   // 强命中：直接作为肯定结果应用
}
else if(score >= 0.70){
   // 弱命中：存在关联，需结合大语言模型分析并输出
}
else {
   // 未命中：拦截请求，抛入兜底或人工闭环流程
}
```

**信息落盘：** 将所有命中情况均记录至流水表 `question_hit_record`中。

## 第十二阶段：答案生成

**数据获取：** 获取精排并命中通过的 `Top 1`，`Top 2`，`Top 3` 提供的内容依据。

- **使用模型：** Qwen3-4B 或 Qwen3-8B（根据服务器算力调整参数规模）
- **核心提示词（Prompt）：**
> 请严格根据提供知识回答。
> 禁止编造。
> 禁止引用知识库外内容。

## 第十三阶段：热点统计

**统计处理：** 按固定周期分析高频提问词（如：`补考`、`毕业`、`选课`），直接写入统计信息表 `stat_hot_question`。通过可视化面板或脚本直接生成相关**教务月报**。

## 第十四阶段：未命中闭环

处理低分和无召回的极端及例外提问：
1. **触发拦截：** 比如问题 `“奖学金评定标准”`，检索出最高分仅为 `0.42`，走未命中分支。
2. **入库记录：** 进入系统异常库 `unrecognized_query`。
3. **后台展示：** 管理员登录后台可见模块：`[查看未命中 Top 20]`。
4. **人工完善：** 管理员审核通过后手动补齐知识库。
5. **归档反馈：** 数据重新切片并向量化进入 Milvus，问答助手系统通过良性循环越来越完善。

---

## 核心模型矩阵归纳

| 实施环节 | 选择大模型 | 功能与目标定位 |
| :--- | :--- | :--- |
| **文档清洗** | Qwen3-0.6B | 将原生文档的文字提炼重组成结构化的 FAQ |
| **问题改写** | Qwen3-0.6B | 将含糊不清的口语化描述规范成正式教务问法 |
| **模块分类** | MacBERT | 基于文本打上标签，缩减 Milvus 扫描范围 |
| **向量生成** | BGE-base-zh-v1.5/small | 实施高维词嵌入，输出768维可对碰向量 |
| **重排序精排** | bge-reranker-large | 提供二次细分精确打分，留下最相关的段落 |
| **最终答案生成**| Qwen3-4B / 8B | 把控知识边界并在给定 Context 中生成答案 |
| **心理咨询接入**| Coze 工作流 | 处理纯感性交互及安抚，交接外部工具工作流 |

# 智能问答助手系统后端重构计划

基于全新的 AI 模型内嵌方案，结合当前的 Spring Boot 后端项目结构，制定以下后端模块化重构计划。
**特别说明**：所有模型均由本系统内嵌直接加载推理，不依赖于 Ollama、VLLM 等外部模型服务。

## 一、 后端重构计划（按模块分类）

建议在您的后端项目中引入以下模块结构调整：

### 1. 知识库管理模块 (Knowledge Base Module)
*负责处理第一、第二、第三阶段（导入、清洗、切片）*
* **路径**: `service/impl/KnowledgeBaseServiceImpl.java`
* **新增核心组件**:
  * `DocumentParserUtil`: 引入 `Apache POI` 和 `PDFBox`，负责解析 Word、Excel、PDF 生成纯文本。
  * `DataCleanService`: 负责加载 **Qwen3-0.6B** 模型执行本地推理，将纯文本长篇大论清洗、重写为 FAQ 结构（Q & A）。
  * `KnowledgeChunker`: 文本分块器，按照语义或基于 FAQ 的单个 Q&A 对进行切片（Chunk）。
  * **落表**: 将切片后的数据存入 `kb_qa_entry` 表。

### 2. 向量检索模块 (Vector Search Module)
*负责第四、第五、第九、第十、第十一阶段（向量化、存库、检索、重排序、命中判定）*
* **路径**: `service/impl/VectorSearchServiceImpl.java`
* **新增核心组件**:
  * `EmbeddingService`: 使用 **ONNX Runtime (Java)** 或 **DJL (Deep Java Library)** 本地加载 **BGE-small-zh-v1.5** 模型文件，将文本转换为 768 维向量。
  * `MilvusClientManager`: 维护与 Milvus 向量数据库的连接，执行 Upsert 和 Search 操作。
  * `RerankService`: 负责获取 Milvus 的 Top-20 结果后，本地加载 **bge-reranker-large** 模型进行二次精排打分，截取 Top-3。
  * `HitRuleEngine`: 封装命中率判断逻辑（`>=0.85` 强命中，`0.70<score<0.85` 弱命中，`<0.70` 未命中）。

### 3. 智能问答核心模块 (AI Chat Module)
*负责第六、第七、第八、第十二阶段（提问接收、问题改写、意图分类、答案生成）*
* **路径**: `service/impl/AiChatServiceImpl.java`
* **新增核心组件**:
  * `QueryRewriteService`: 接收学生口语化提问，本地调度 **Qwen3-0.6B** 进行规范化改写。
  * `IntentClassifierService`: 本地加载 **MacBERT**，对改写后的问题进行二级模块分类推理（补考、重修等），限制检索范围。
  * `AnswerGeneratorService`: 整合 Top-3 检索内容与用户问题，组装 Prompt，使用 DJL 工具包加载 **Qwen3-4B / 8B**，流式（WebSocket/SSE）生成最终答案并返回给前端。

### 4. 统计与兜底闭环模块 (Stats & Fallback Module)
*负责第十三、第十四阶段（热点统计、未命中闭环）*
* **路径**: `service/impl/DataStatServiceImpl.java` & `service/impl/UnrecognizedQueryServiceImpl.java`
* **核心组件**:
  * 热词统计定时任务（`@Scheduled`）：定时扫描提问表，生成报表并写入 `stat_hot_question`。
  * 意图未命中拦截器：对于得分 `< 0.70` 的请求，阻断交给大模型生成，进入 `unrecognized_query` 表，打上“待人工处理”标签。

## 二、 AI模型放置位置及管理建议

建议不要将几 GB 至十几 GB 的模型权重文件放在代码仓库（如 `src`）下，避免撑爆 Git。

**推荐放置路径：**
在服务器或本地电脑的统一外挂目录下，例如在项目根目录同级创建一个 `models/` 目录，或者直接使用操作系统的 `/opt/aigc-models/`（Linux/Mac）或 `D:\aigc-models\` (Windows)。

**目录结构示例：**
```text
/opt/aigc-models/ (或者项目根目录下的 /models/)
  ├── qwen3-0.6b/           # 存放 config.json, model.safetensors 等
  ├── qwen3-4b/
  ├── bge-small-zh-v1.5/
  ├── bge-reranker-large/
  └── macbert-base/
```

在 `application-secrets.properties` 或 `application.properties` 中配置该路径：
```properties
# AI 模型根目录配置
ai.models.base-path=/opt/aigc-models/
```

## 三、 系统服务启动时自动检测与安装（跳过已存在模型）

为了实现“系统启动时检测模型，若不存在则自动下载安装”，可以通过实现 `ApplicationRunner` 接口，配合 Python 脚本（利用 ModelScope SDK）来优雅地完成下载动作。
*(注意：在下载完成后，由于本系统采取直接内嵌调用的方式，您需要在 `pom.xml` 中引入 `DJL (Deep Java Library)` 或 `ONNX Runtime` 的依赖集，由它们来加载 `models/` 目录下的模型权重文件执行推理)*

### 1. 准备模型下载 Python 脚本
将下方内容保存至：`backend/scripts/download_models.py`
```python
import sys
import os
from modelscope import snapshot_download

def download_if_not_exists(model_id, local_dir):
    if not os.path.exists(local_dir):
        print(f"Directory {local_dir} not found. Start downloading {model_id}...")
        snapshot_download(model_id, local_dir=local_dir)
        print("Download complete.")
    else:
        print(f"Model {model_id} already exists in {local_dir}. Skipping.")

if __name__ == "__main__":
    model_id = sys.argv[1]    # 例如: qwen/Qwen2.5-0.5B
    local_dir = sys.argv[2]   # 例如: /opt/aigc-models/qwen3-0.6b
    download_if_not_exists(model_id, local_dir)
```

### 2. 在后端项目中增加模型启动检测器 (Java部分)
将下方内容保存至：`backend/src/main/java/me/rainhouse/qasystem/config/ModelInitializationRunner.java`

利用 Spring Boot 在服务启动时自动扫描本地磁盘：

```java
package me.rainhouse.qasystem.config;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Value;
import org.springframework.boot.ApplicationArguments;
import org.springframework.boot.ApplicationRunner;
import org.springframework.stereotype.Component;

import java.io.BufferedReader;
import java.io.File;
import java.io.InputStreamReader;
import java.util.Map;
import java.util.HashMap;

@Component
public class ModelInitializationRunner implements ApplicationRunner {

    private static final Logger log = LoggerFactory.getLogger(ModelInitializationRunner.class);

    @Value("${ai.models.base-path:/opt/aigc-models/}")
    private String modelBasePath;

    @Override
    public void run(ApplicationArguments args) throws Exception {
        log.info("=========== 开始检测 AI 模型完整性 ===========");
        
        // 声明系统需要的模型及其对应的远程库ID (以ModelScope为例)
        Map<String, String> requiredModels = new HashMap<>();
        requiredModels.put("qwen3-0.6b", "qwen/Qwen-0.6B"); // 请按需填入真实的魔搭仓库ID
        requiredModels.put("bge-small-zh-v1.5", "BAAI/bge-small-zh-v1.5");
        requiredModels.put("macbert-base", "hfl/macbert-base");
        
        File baseDir = new File(modelBasePath);
        if (!baseDir.exists() && !baseDir.mkdirs()) {
            throw new RuntimeException("无法创建模型存放目录: " + modelBasePath);
        }

        for (Map.Entry<String, String> entry : requiredModels.entrySet()) {
            String dirName = entry.getKey();
            String repoId = entry.getValue();
            File modelDir = new File(baseDir, dirName);

            if (modelDir.exists() && modelDir.list() != null && modelDir.list().length > 0) {
                log.info("模型 [{}] 已存在，路径: {}, 跳过下载", dirName, modelDir.getAbsolutePath());
            } else {
                log.info("模型 [{}] 缺失，正在触发自动下载环境...", dirName);
                downloadModelViaPython(repoId, modelDir.getAbsolutePath());
            }
        }
        log.info("=========== AI 模型全量检测完毕 ===========");
    }

    private void downloadModelViaPython(String repoId, String targetDir) {
        try {
            // 调用外部的下载脚本
            String scriptPath = "scripts/download_models.py"; 
            ProcessBuilder pb = new ProcessBuilder("python3", scriptPath, repoId, targetDir);
            pb.redirectErrorStream(true);
            Process process = pb.start();
            
            // 实时打印下载进度到 Java 日志中
            try (BufferedReader reader = new BufferedReader(new InputStreamReader(process.getInputStream()))) {
                String line;
                while ((line = reader.readLine()) != null) {
                    log.info("[下载器]: {}", line);
                }
            }
            int exitCode = process.waitFor();
            if (exitCode != 0) {
                log.error("模型 {} 下载失败，状态码: {}", repoId, exitCode);
            }
        } catch (Exception e) {
            log.error("触发模型自动下载时发生异常", e);
        }
    }
}
```

**运行逻辑：**
1. 在运行您的 Java 服务时，Spring Boot 会在启动的末尾阶段执行 `ModelInitializationRunner`。
2. 程序拉取配置的存放路径目录，检查设定的对应模型文件夹是否存在以及是否有内容。
3. 若存在文件，程序直接跳过下载并成功运行服务。
4. 若文件夹缺失内容，会自动触发后台 Python 脚本 `download_models.py`，拉取对应权重到指定目录。拉取完毕后提供问答服务。

# 智能问答辅助系统 — 前端功能文档 v0.3

> **变更说明**: v0.3 在 v0.2 基础上，结合数据库设计文档 `backend/llm_qa_sy.sql`，修正了角色定义（从 5 种推测收敛为 DB 中明确的 3 种）、补充了各实体的状态枚举及字段细节，并将数据库表结构与前端页面字段逐表对齐。

---

## 1. 角色定义

> **数据来源**: `sys_user.role` — `tinyint(4)`: `1`-学生, `2`-教师, `3`-管理员

| 角色 | DB 值 | 说明 |
|------|-------|------|
| 学生 | `1` | 核心服务对象，通过 AI 对话获取学业、心理、校园生活等方面的智能问答服务 |
| 教师 | `2` | 涵盖客服坐席、辅导员、教务老师等职能。具体权限由后端接口层做功能级控制，前端按路由守卫 + 功能按钮显隐实现 |
| 管理员 | `3` | 最高权限，可访问全部管理功能、管理用户权限与系统配置 |

> **说明**: v0.2 曾推测 5 种角色（客服/辅导员/教务老师/管理员），经数据库确认实际仅 `学生/教师/管理员` 三种。客服、辅导员、教务老师均为"教师"角色下的业务职能子集，前端不做角色级强制隔离，而是根据后端接口返回的权限位或业务数据做按钮级显隐控制。

### 用户表字段 (`sys_user`)

| 字段 | 类型 | 前端用途 |
|------|------|----------|
| `id` | bigint | 用户唯一标识，API 请求时传入 |
| `username` | varchar(50) | 登录账号 |
| `password` | varchar(100) | 密码（SSO 登录时可为空） |
| `real_name` | varchar(50) | 真实姓名，展示于个人中心、客服台 |
| `role` | tinyint(4) | 角色路由依据：1→学生端，2/3→管理端 |
| `phone` | varchar(20) | 联系电话 |
| `campus_sso_id` | varchar(100) | SSO 唯一标识，一网通登录关联 |
| `wechat_openid` | varchar(100) | 微信公众号 OpenID，微信登录关联 |
| `avatar_url` | varchar(255) | **专属形象/头像** URL，对话页 + 个人中心展示 |

## 3. 通用功能（所有角色）

### 3.1 登录认证

| 功能点 | 对应 API | 关联字段 |
|--------|----------|----------|
| 账号密码登录 | `POST /api/auth/login` | `username`, `password` |
| 一网通 SSO 登录 | `POST /api/auth/sso/callback` | `campus_sso_id` |
| 微信 OAuth 登录 | `GET /api/wechat/login` | `wechat_openid` |
| 获取用户信息 | `GET /api/auth/info` | 返回 `SysUser` 全字段（密码脱敏） |
| 登出 | `POST /api/auth/logout` | — |

**页面内容**：
- 登录表单（用户名 + 密码）
- "一网通登录" 按钮 → 跳转学校统一认证页面
- "微信登录" 入口 → 展示公众号二维码
- 登录失败提示、Token 过期自动跳转登录页
- 根据 `role` 字段：`1`→学生端 `/student/chat`，`2`/`3`→管理端 `/admin/dashboard`

### 3.2 全局设计规范

| 要点 | 说明 | DB 关联 |
|------|------|---------|
| 专属 IP 形象 | 通过 `sys_user.avatar_url` 加载定制虚拟形象，统一出现在对话页、加载态、欢迎页 | `avatar_url` |
| 多端响应式适配 | 学生端需同时适配 PC Web、手机浏览器、微信公众号内嵌页面三端 | — |
| 情感化交互 | 加载时显示形象动画、问候语个性化（分时段）、空状态趣味提示 | — |
| 数据脱敏展示 | 学生 `student_profile.masking_id` 代替真实学号传给 Coze；管理端列表中对敏感字段做掩码 | `masking_id` |

## 4. 学生端功能

### 4.1 AI 智能问答对话（学生端核心）

**对应 API**：

| 接口 | 路径 |
|------|------|
| 发送文本消息 | `POST /api/chat/text` |
| 发送语音消息 | `POST /api/chat/voice` |
| 获取推荐问题 | `GET /api/chat/suggested-questions` |
| 申请转人工 | `POST /api/chat/transfer-to-human` |
| 上报未识别回答 | `POST /api/chat/report-unrecognized` |
| 获取座机推送 | `GET /api/business/phone`（由 `biz_contact` 表提供数据） |

**关键数据模型**：

```
chat_session:  id | user_id | status (0-AI托管, 1-转人工, 2-已结束) | admin_id
chat_message:  id | session_id | sender_type (1-用户, 2-AI, 3-人工客服) | msg_type (1-文本, 2-语音)
                 | content | media_url | intent_tag | created_at
biz_contact:   id | biz_module | teacher_name | phone_number
```

**页面内容**：

- **顶部栏**: 专属 IP 形象（`avatar_url`）+ 系统名称
- **对话区**:
  - 聊天气泡按 `sender_type` 区分样式：用户（右，蓝色）、AI（左，灰色）、人工客服（左，绿色）
  - 滚动加载历史 `chat_message` 记录
  - AI 回答中显示"正在输入..."动画（专属形象动效）
  - 语音消息 (`msg_type=2`) 显示可播放的音频条，点击播放 `media_url`
  - AI 回答底部展示 `intent_tag` 意图标签（如"已匹配：选课咨询"）
  - 答案底部提供"有帮助/无帮助"评价按钮
- **推荐问题区**（欢迎态）:
  - "猜你想问" 气泡列表 → `GET /api/chat/suggested-questions`（底层查 `stat_hot_question`）
- **业务关联推送卡片**（对话中动态出现）:
  - AI 识别到特定 `intent_tag` 时，查 `biz_contact` 表推送对应业务的教师座机
  - 卡片：图标 + `biz_module` + `teacher_name` + `phone_number`
- **输入区**（底部固定）:
  - 文本输入框 + 发送按钮 → `POST /api/chat/text`
  - 麦克风按钮 → 录音 → `POST /api/chat/voice`，`media_url` 存储录音文件
- **转人工按钮** → 调 2.4，`chat_session.status` 变为 `1`
  - 排队状态展示，人工接入时切 WebSocket 实时对话
- **点踩/反馈按钮** → 调 2.8，写入 `unrecognized_query` 表

#### 4.1.1 人工转接 WebSocket 实时对话

| 要点 | 数据流 |
|------|--------|
| 状态机 | `chat_session.status`: `0`(AI) → `1`(转人工) → `2`(已结束) |
| 消息区分 | `chat_message.sender_type`: 用户(`1`) / AI(`2`) / 人工客服(`3`) |
| 前端展示 | 等待中（status=1 且无 admin_id）→ 接待中（status=1 且有 admin_id）→ 已结束（status=2） |
| WebSocket | 双向收发，前端即时渲染 `content` 到对话区 |
| 断线重连 | 自动重连 + 提示 |

### 4.2 心理倾诉

**对应 API**：`POST /api/chat/psychological`

**关联数据**：`student_profile.psychological_tag`（画像中的心理标签，脱敏后传给 Coze Agent 做个性化安抚）

**页面内容**：
- 柔和 UI 色调，对话形式
- AI 返回共情话术（Coze 心理倾诉 Agent 生成）
- 压力等级指示器（若后端返回 `risk_level` 相关内容）
- 隐私提示条："对话内容严格保密，不做个人身份关联"（Coze 端仅接收 `masking_id`）
- "需要进一步帮助" → 引导至学校心理咨询中心

### 4.3 个性化资源推荐

> 基于 `student_profile` 画像标签（`psychological_tag`、`risk_level`、GPA 等脱敏特征）由 Coze Agent 匹配推荐资源

**页面内容**：
- 资源卡片列表：学习资料 / 辅导活动 / 心理调适资源
- 每卡片：标题 + 摘要 + 类型标签 + 查看详情
- "已读/已收藏"标记

### 4.4 我的成长档案

> 数据来源：`student_growth_archive` + `academic_warning_record`

| DB 字段 | 前端展示 |
|---------|----------|
| `create_time` | 时间轴节点时间 |
| `warning_level` | 预警级别标签 |
| `survey_indicator` | 调查问卷指标变化 |
| `help_plan` | Coze 生成的一对一帮扶方案 |

**页面内容**：
- **时间轴**: 按学期纵向排列关键节点（预警、帮扶、指标变化）
- **小型图表**: 学分完成环形图、成绩变化折线图
- 数据均为脱敏后仅本人可见

### 4.5 个人中心

- 头像 (`avatar_url`) + 基本信息 (`real_name`, `username`)
- 修改密码入口
- "我的反馈"列表：历史上报的 `unrecognized_query` 及处理状态
- 退出登录

## 5. 管理端功能

### 5.1 数据看板（管理端首页）

**数据来源**：`stat_hot_question`（`question_text` + `frequency` + `stat_date`）

**页面内容**：

**5.1.1 图表区**
- 词云图：按 `frequency` 权重渲染关键词
- 提问趋势折线图：按 `stat_date` 聚合
- 业务分类柱状图：按 `chat_message.intent_tag` 分组统计
- Top 问题排行榜

**5.1.2 筛选**
- 时间范围（7/14/30 天 / 自定义 `stat_date` 区间）、业务模块筛选

**5.1.3 摘要卡片**
- 总提问数、活跃用户数、未处理问题数（`unrecognized_query WHERE status=0`）、今日转人工次数

**5.1.4 主动推送配置**
- 管理员选择高频 `stat_hot_question` 记录 → 推送至学生端"猜你想问"或跑马灯公告

### 5.2 知识库管理

**数据来源**：`kb_document` + `kb_qa_entry`

| 表 | 关键字段 | 前端用途 |
|----|----------|----------|
| `kb_document` | `file_name`, `process_status` (0-待解析/1-解析中/2-成功/3-失败), `uploader_id` | 批量导入文件追踪 |
| `kb_qa_entry` | `question`, `answer`, `status` (0-禁用/1-启用), `document_id`, `created_by` | 词条 CRUD |

**页面内容**：

**5.2.1 词条列表**
- 表格列：`question`、`answer`摘要、`status`（启用/禁用开关）、来源文档（`document_id`）、`created_at`
- 搜索 + 状态筛选（`status`：全部/启用/禁用）
- 操作：编辑、删除、启用/禁用切换

**5.2.2 新增/编辑词条**
- 表单：问题输入框 + 答案富文本编辑器
- 提交 → `POST /api/kb/entries`

**5.2.3 批量导入**
- 上传 `.xlsx` / `.docx` → `POST /api/kb/upload`
- 导入后生成 `kb_document` 记录，`process_status` 从 `0` → `1` → `2`/`3`
- 前端展示解析进度（轮询或 WebSocket 推送 `process_status` 变化）
- 解析结果预览：成功 N 条（入库为 `kb_qa_entry`）、失败 M 条（含失败原因）

### 5.3 人工客服台

**数据来源**：`chat_session` + `chat_message`

| 字段 | 用途 |
|------|------|
| `chat_session.status` | `0`-AI托管(不显示在客服台), `1`-转人工(待接入/接待中), `2`-已结束 |
| `chat_session.admin_id` | 匹配当前客服，判断"我的会话" |
| `chat_message.sender_type` | `1`-用户, `2`-AI(历史), `3`-人工客服 |
| `chat_message.intent_tag` | 展示 AI 识别的问题意图，辅助客服判断 |

**页面内容**：

**会话列表（左侧栏）**
- Tab：待接入（`status=1 AND admin_id IS NULL`）/ 接待中（`status=1 AND admin_id=我`）/ 已结束（`status=2`）
- 每项：学生匿名标识、`intent_tag` 问题摘要、等待时长
- 新消息提醒（橙色圆点）
- "接入"按钮 → 写入 `admin_id`

**对话区（右侧）**
- 历史消息按 `sender_type` 区分气泡样式
- AI 阶段与人工阶段用分隔线标识
- 底部输入框 + 快捷回复短语
- WebSocket 实时收发

**操作区**
- "结束会话" → `POST /api/chat/admin/finish` → `status` 变为 `2`
- 会话备注（内部可见）
- 统计：当前接待数 / 总排队数

### 5.4 未识别问题管理

**数据来源**：`unrecognized_query`

| 字段 | 前端展示/操作 |
|------|--------------|
| `query_text` | 问题原文（表格列） |
| `user_id` | 提问学生（可关联查看，脱敏展示） |
| `status` | `0`-待处理 / `1`-已入知识库 / `2`-忽略 |
| `created_at` | 提问时间 |

**页面内容**：
- **列表**: 分页，列：`query_text`、`created_at`、`status`
- **状态筛选**: `0`/`1`/`2` 三态 Tab
- **操作**:
  - `status: 0→1`："添加到知识库" → 弹窗预填 `query_text`，编辑答案后入库为 `kb_qa_entry`
  - `status: 0→2`："忽略"
- **统计卡片**: 待处理数 (`COUNT WHERE status=0`)、本月新增、已处理
- **分页器**

### 5.5 学业帮扶

**核心数据模型**：

```
student_profile:           user_id | masking_id | gpa | failed_courses_cnt
                           | psychological_tag | risk_level (0-无/1-低/2-中/3-高)

academic_warning_record:   student_id | term | warning_reason (AI分析)
                           | ai_suggested_plan (AI方案) | report_pdf_url

student_growth_archive:    real_student_id | fake_student_id (给Coze的假ID)
                           | warning_level | survey_indicator | help_plan

v_student_academic_profile: 视图，关联 sys_user + student_profile (WHERE role=1)
```

#### 5.5.1 学业预警与策略生成

**页面内容**：
- **触发表单**: 学生选择器（来源 `v_student_academic_profile` 或直接查学生）+ 学期 → `POST /api/academic/generate-warning`
- **结果展示区**：
  - 学生概况卡：`gpa`、`failed_courses_cnt`、`risk_level`（红/黄/绿）
  - AI 预警分析：`warning_reason`
  - AI 帮扶方案：`ai_suggested_plan`（原因分析 → 建议措施 → 推荐资源）
  - "保存为帮扶记录" → 写入 `academic_warning_record`
- **批量操作**: 多选学生批量生成预警

#### 5.5.2 学生画像管理

**页面内容**：
- **搜索**: 按学号/姓名（含脱敏）搜索
- **画像展示**:
  - `gpa`、`failed_courses_cnt`（学业维度）
  - `psychological_tag`（心理维度 — 仅脱敏展示）
  - `risk_level`：0 无风险 / 1 低 / 2 中 / 3 高，颜色标识
  - `masking_id`：展示脱敏 ID（即传给 Coze 的假 ID）
- **编辑**: 单个编辑表单或批量导入 → `POST /api/academic/profile/update`
- **注意**: `real_student_id` vs `fake_student_id` 分离 — 管理端列表可用真实学号检索，但传给 Coze 时使用 `fake_student_id` / `masking_id`

#### 5.5.3 成长档案与帮扶报告

**页面内容**：

**帮扶记录列表**:
- 表格列：学生信息、`term`、`created_at`、`warning_reason` 摘要、`risk_level`
- 筛选：按学期、风险等级

**成长档案详情**（单条记录）:
- 学生成长时间轴（`student_growth_archive.create_time` + `warning_level` + `survey_indicator`）
- 各学期关键指标对比
- AI 帮扶方案全文 (`help_plan`)
- "生成 PDF 报告" → `POST /api/academic/generate-pdf-report` → `report_pdf_url` 回写

**学院成效报告**:
- 按学期/学院维度汇总
- ECharts: 帮扶人数、预警解除率、前后 GPA 对比
- "导出学院成效报告 PDF"

| `/admin/profile` | 管理端个人中心 | | ● | ● |

### 6.2 鉴权策略

- **路由守卫**: 未登录 → `/login`；`role=1` 访问 `/admin/*` → 重定向 `/student/chat` + Toast
- **Token 管理**: localStorage → Axios 拦截器注入 `Authorization` Header
- **401 处理**: 清除 Token → `/login`
- **按钮级控制**: 删除词条、系统配置等敏感操作仅 `role=3` 可见
